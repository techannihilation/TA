# Tech Annihilation ![CI Status](https://img.shields.io/github/actions/workflow/status/techannihilation/TA/validate-game.yml?branch=master)

Uses [Recoil Engine](https://github.com/beyond-all-reason/RecoilEngine), fork of [Spring Engine](https://springrts.com/).
This Mode looks like to Balanced Annihilation with more units and a morph system.

## Getting Started

Nixtux and Finky started to work on the mode in 2010 on github.
The game uses a rapid tool with springlobby. All commits update the game in real time.

### Prerequisites & Installing

To install the game, 3 options:  
  
  1. Download [skylobby of tizbac](https://github.com/tizbac/skylobby/releases) . then you will find a battleroom with Tech-annihilation mention. You can enter in the battleroom and maps and last version will be automatically download.  
  2. Use [start.sh](start.sh) on Linux, might work on Windows with [msys2](https://medium.com/@saml/setting-up-a-light-weight-linux-like-non-wsl-terminal-environment-on-windows-59a583c958ed) this is untested please report back.  
     Consult: `./start.sh --help` most likely you want `./start.sh run <directory-to-run-in>`
  3. Download the [Recoil Engine](https://github.com/beyond-all-reason/RecoilEngine/releases). And Download this repository and paste on spring/games/TA.SDD folder. You could play alone to test the mode.

## Versioning

Actually, the Tech annihilation mode, have no version. All commits done are automatically uploaded on [Rapid](http://tard.repo.springrts.com/builds/).
